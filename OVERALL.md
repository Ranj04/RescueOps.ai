RescueOps HQ — an agent harness you can watch, test, and text.
The premise: a model is just a brain. It becomes an agent only when there's an operating system around it — memory, context management, safety controls, and tool access. That's the harness, and it's what most agent demos skip. This project makes the harness the product, using incident response as the domain that proves it.
How it works, top to bottom:
An incident opens (a production outage, or a SOC alert in the security domain pack). An Incident Commander — an LLM agent — dispatches five specialists: Triage → Diagnosis → Remediation → Verification → Postmortem. The Commander makes real decisions (fast-path vs. deep diagnosis, retry vs. escalate) but only from legal moves defined in a policy.json file that code enforces at runtime. If the LLM tries something illegal, it gets overruled and the overrule is logged. Risky fixes can't execute without human approval — the policy forces it, not a prompt.
Every action any agent, tool, or human takes appends one event to an append-only event stream. That stream is the spine of the system: the dashboard, the animated Ops Floor (little SVG responders working at desks, freezing when they need approval, catching fire under chaos), the audit log, and the eval harness are all just different renderers of the same real events. Nothing on screen is faked.
The human gate reaches the human where they actually are: when approval is needed, the on-call engineer gets a text, replies YES or NO, and the run resumes — same state machine as the web approval panel, first response wins. When the incident resolves, a summary report lands on the same phone.
The whole thing runs on EdgeOne Makers — their agent runtime hosts the crew, every model call rides their gateway (with live failover), paused pipeline state persists in their session store, their KV holds the event log, their tracing links from every event row — and deploys in one click.
What makes it win: the trust layer on top. A chaos console lets anyone — including a judge — kill data sources or the primary model mid-incident and watch it degrade gracefully. A ground-truth eval scores measured accuracy against labeled incidents. Everyone at the event will deploy an agent; this is the team that attacks its own agent on stage and shows the receipts.
